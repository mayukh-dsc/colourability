# colourability — Development Plan

## Project Overview

**colourability** is a zero-dependency, open-source TypeScript module that applies color-blindness filters and a grayscale filter to any webpage. It uses a single uniform mechanism (SVG `feColorMatrix`) and works across mobile, desktop, and large-format screens (TVs, kiosks), correctly handling HTML video, canvas, WebGL, and static content.

---

## Goals

- Pluggable on any website via `<script>` tag or npm import
- No runtime dependencies
- Tiny footprint (target: < 4 KB minified + gzipped)
- Works over `<video>`, `<canvas>`, WebGL content via a single SVG filter mechanism
- Responsive to any screen size out of the box
- Color-blindness simulation (all 3 types) and daltonization (all 3 types)
- Grayscale filter
- Minimal public API — no `register` / events in v1
- Published to npm under MIT license

## Technical Decisions

### Why a single mechanism (SVG `feColorMatrix`)?

All 7 filters — grayscale, simulation, and daltonization — are 4×5 linear matrix operations. `feColorMatrix` expresses them exactly with no branching or special-casing. This removes the need for an overlay `<div>`, CSS filter shorthand strings, blend modes, and z-index management.

`color-interpolation-filters="linearRGB"` on the `<filter>` element causes the browser to apply the matrix in linear light space (after gamma decoding), which is required for perceptually correct color-blindness matrices.

### Why no `src/filters.ts`?

Preset names map to matrix strings via `FILTERS` in `src/matrix.ts`. A separate file would only duplicate imports from `matrix.ts` with no extra semantics. `supportsIntensity` is always true for every preset — no per-preset flags needed.

### How Daltonization Works

The Fidaner algorithm collapses to a single linear matrix:

```
M_daltonize = I + (I − M_simulate) × M_shift_error
```

Because both `M_simulate` and `M_shift_error` are linear, the full algorithm is a single matrix multiplication — expressible as one `feColorMatrix`. Pre-computed matrices live in `src/matrix.ts`. `setIntensity(t)` interpolates each coefficient between the identity matrix (t=0) and the full matrix (t=1), enabling partial correction for anomalous trichromacy.

### Known Limitations

| Limitation | Impact | Mitigation |
|---|---|---|
| `filter: url(...)` on `<html>` creates a new stacking context | `position:fixed` elements on host page may reposition | v2: optional `safeMode` or alternative target — not v1 |
| Cross-origin iframes | Filter does not affect inner content | Browser security boundary — intentional |
| Daltonization matrices are pre-computed approximations | Not as precise as a full LMS runtime algorithm | `linearRGB` interpolation corrects gamma; LMS matrix is folded in. Accuracy matches Chrome DevTools and OS accessibility tools. |

### Module Size Budget
- Source: < 8 KB
- Minified: < 3 KB
- Minified + gzipped: < 1.5 KB
